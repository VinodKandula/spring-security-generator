About Spring Security Generator

I like the Spring Initializr project for creating spring projects, and was interested whether a menu driven approach could be applied to Spring Security.

Security requirements are often simple to define(block or allow access to this directory, validate credentials, ...), but hard to implement. The idea of this approach is to set out the requirements, and autogenerate the code, and dependencies. It also has the advantage of allowing best practice to be incorporated into the generated code.

The generator is written in clients-side Javascript. I think this is important in allowing users of the site confidence to know that their configuration is not being shared with some unknown third-party. The project is also on github -
